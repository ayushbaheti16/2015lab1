    \subsubsection{Pythons and ducks}\label{pythons-and-ducks}

Notice that we used the series in the x-list and y-list slots in the
\texttt{scatter} function in the \texttt{plt} module.

In working with python I always remember: a python is a duck.

What I mean is, python has a certain way of doing things. For example
lets call one of these ways listiness. Listiness works on lists,
dictionaries, files, and a general notion of something called an
iterator.

A Pandas series plays like a python list:

    \begin{Verbatim}[commandchars=\\\{\}]

\end{Verbatim}


    We can construct another list by using the syntax below, also called a
list comprehension.

    In other words, operations on numpy arrays, and by extension, Pandas
Series, are \textbf{vectorized}. You can add two numpy lists by just
using \texttt{+} whereas the result isnt what you might expect for
regular python lists. To add regular python lists elementwise, you will
need to use a loop:

    \begin{Verbatim}[commandchars=\\\{\}]

    \textbf{Vectorization} is a powerful idiom, and we will use it a lot in
this class. And, for almost all data intensive computing, we will use
numpy arrays rather than python lists, as the python numerical stack is
based on it.

You have seen this in idea in spreadsheets where you add an entire
column to another one.
